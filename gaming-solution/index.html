场景 1｜新用户注册迎新 + 身份认证引导

新用户激活 全球新兴市场通用 再次进入：不可再次进入
旅程编排示意图
业务痛点
全球游戏平台注册→身份认证完成率普遍仅 45–65%。 未完成认证的用户无法使用充值、提现等核心功能，直接影响付费转化链路。 用户注册后的前 24 小时是引导完成认证的最佳窗口。
旅程组件配置

  • 提前退出：事件"身份认证完成"→ 自动退出旅程
  • 转化统计：事件"身份认证完成"，监听 1 天
  • 第一步发送：立即发送欢迎消息 + 平台介绍，引导完成认证
  • 等待事件 × 2：10min / 12h 逐级等待"身份认证完成"——先观察自然转化，未完成再触达

场景 2｜首充激活 · 3 日黄金期

首次付费转化 再次进入：不可再次进入
旅程编排示意图
业务痛点
注册用户中仅有 3–12% 会完成首次充值（行业平均）。完成身份认证后的 3 日内是首充转化的黄金窗口—— 超过 3 天未充值，后续转化率将大幅下降。精准的时机触达可显著提升首充转化。
旅程组件配置

  • 提前退出：事件"充值成功"→ 自动退出旅程
  • 转化统计：事件"充值成功"，监听 1 天
  • 收入统计：事件"充值成功"，金额属性 recharge_amount，监听 1 天
  • 第一步：认证成功即时推送首充礼包活动通知
  • 等待事件 × 2：10min / 24h 内等待"充值成功"，超时走下一触达
  • 可发送时间：终端时区 9:00–21:00（第 2 轮起加入）
  • 再次进入：不可再次进入
可参考指标
同类游戏平台首充激活旅程，Push 点击率 18%，首充转化率 7%，3 日黄金期内首充率预期提升 8–12%。

场景 3｜充值未完成挽回

付费转化 再次进入：1 小时后，不限次数
旅程编排示意图
业务痛点
玩家点击充值按钮或访问充值页面后，因网络波动、支付流程复杂、临时分心等原因中途放弃， 是平台付费流失的高频场景。及时的提醒与活动推送可显著提升充值完成率。
旅程组件配置
  • 开始组件：事件"点击充值按钮"或"访问充值页面" + 属性: 已完成身份认证 触发进入
  • 提前退出：事件"充值成功"→ 自动退出旅程
  • 转化统计：事件"充值成功"，监听 2 小时
  • 收入统计：事件"充值成功"，金额属性 recharge_amount，监听 2 小时
  • 等待事件：5 分钟内等待"充值成功"——先观察是否自然完成
  • App Push：充值活动提醒 + 一键跳转充值页（deep link）
  • 等待事件：1 小时内等待"充值成功"，超时发 WhatsApp
  • WhatsApp：充值礼包限时活动提醒
  • 再次进入：1 小时后可再次进入，不限制次数
可参考指标
充值未完成场景 5 分钟内触发 Push，点击率 11%，挽回转化率 10%；Web Push 点击率 10%，转化率 9%，整体充值完成率提升 15–25%。

场景 4｜沉睡玩家 3 级唤醒（3 / 7 / 30 日）

3日留存 / 召回 再次进入：30 分钟后，不限次数
旅程编排示意图
业务痛点
3 日留存（Day 3）是游戏平台最关键的早期留存节点之一。 用户在注册后 72 小时内若未主动回访，极大概率不会成为付费用户。 游戏平台存量用户中保守估计 40% 处于 30 日沉睡状态， 而阻止用户在 3 日内流失，通常比长期沉默后再召回的综合成本更低（具体倍数因渠道成本与客单而异，此处不作固定承诺）。 分阶段、差异化的召回触达——结合专属回归礼包、新活动预告、限时奖励—— 是在 3 日窗口内拉回流失玩家的最有效方式。
旅程组件配置
  • 开始组件：属性"最后登录时间距今超过 3 天（72h–73h 精确窗口）"触发进入
  • 提前退出：事件"登录"→ 自动退出旅程
  • 转化统计：事件"登录"，监听 5 天
  • 第 3 天：App Push 发送"好久不见"召回消息 + 回归礼包（3 日留存关键触点）
  • 等待事件：1 天内等待"登录"，超时继续触达
  • 第 4–6 天：每日 App Push 逐步升级内容（新活动 → 专属奖励 → 最后机会）
  • 可发送时间：终端时区 9:00–21:00，每轮 Push 之前加入
  • 再次进入：30 分钟后可再次进入，不限制次数
可参考指标
沉睡唤醒旅程 App Push 点击率 2%，回登转化率 20%；Web Push 点击率 3%，转化率 10%，沉睡用户召回率预期提升 20–40%。

场景 5｜VIP 等级提升通知

会员运营 再次进入：5 分钟后，不限次数
旅程编排示意图
业务痛点
玩家升级 VIP 是高度情绪化的时刻——即时的祝贺与福利推送，能将升级喜悦转化为继续充值的动力。 不同 VIP 阶梯（黄金/铂金/钻石）应有差异化的文案与激励策略， 特别是在即将跨越阶梯时给予更强烈的提示。
旅程组件配置
  • 开始组件（黄金 VIP 段）：属性: VIP 等级 ≥ 11，≤ 16 触发进入
  • 监听事件（多分支）：事件"VIP 等级提升"，多路分流
  • 分支 1：VIP 等级 = 12（即将跨入铂金）→ 发送"再差 1 级即可晋升铂金 VIP"强激励消息
  • 分支 2：VIP 等级 13–16 → 发送"恭喜达到 N 级，领取升级福利"常规通知
  • 转化统计：事件"充值成功"，监听 3 天
  • 再次进入：5 分钟后可再次进入，不限制次数
可参考指标
VIP 升级通知 Push 点击率 7%，配合"临近下一阶梯"差异化文案，付费深度预期提升 6–10%。

场景 6｜每周消费返利提醒

留存激励 再次进入：18 小时后（每周触发）
旅程编排示意图
业务痛点
平台每周向符合条件的玩家发放消费返利奖励，但大量玩家因未收到提醒而错过领取， 导致返利激励机制失效、玩家粘性降低。及时提醒并在到期前再次推送， 能显著提升返利领取率与平台互动频次。
旅程组件配置
  • 开始组件：属性"上周返利领取金额 ≥ 10"触发进入；时间窗口：每周一 9:00 发布
  • 分支 1（未领取提醒）：周一发送领取通知 → 等待事件 1 天内等待"领取返利" → 超时发 Push 提醒 → 再等 4 天 → 超时发 SMS"返利即将过期"
  • 分支 2（已领取后预告）：事件"领取返利"发生 → 周六 9:00 发可发送时间 → 发送"下周记得领取返利"预告消息
  • 转化统计：事件"领取返利"，监听 5 天
  • 再次进入：18 小时后可再次进入（下周一满足条件后自动进入）
可参考指标
周返利提醒 App Push 点击率 5.3%，领取转化率 25%；Web Push 点击率 3.2%，转化率 20%，D7 留存预期提升 8%。

场景 7｜日常签到唤醒提醒

次日留存 / 日活 再次进入：2 分钟后，不限次数
旅程编排示意图
业务痛点
次日留存（Day 1）是最早、成本最低的留存干预窗口。 行业平均次日留存率仅 25–40%——超过 60% 的新用户在注册后第 2 天就再未回来。 签到奖励是建立用户回访习惯最有效的机制之一，但依赖用户主动记起。 在玩家每次登录后约 23 小时精准发送签到提醒， 可将次日登录率显著提升，并为 3 日留存打好基础。
旅程组件配置
  • 开始组件：事件"用户登录"触发进入（每次登录均可触发）
  • 等待：23 小时（在次日回访窗口前发出提醒）
  • 可发送时间：终端时区 9:00–21:00，避免深夜打扰
  • App Push：签到活动提醒，带奖励预告（如"今日签到得 50 游戏币，连续 7 天解锁双倍奖励"）
  • 转化统计：事件"签到成功"，监听 1 天（即次日回访转化）
  • 再次进入：2 分钟后可再次进入，不限制次数（每次登录都可重新触发）
可参考指标
签到提醒 App Push 点击率 2%，签到转化率 10%；Web Push 点击率 1.5%，转化率 8%，D1 留存预期提升 5–10%。

场景 8｜邀请好友达标奖励通知

裂变增长 再次进入：2 分钟后，不限次数
旅程编排示意图
业务痛点
邀请好友是游戏平台低成本获客的核心渠道，但玩家常因未收到进度提醒而失去邀请动力。 在即将达标时给予预警推送、在达标时立即通知领奖，能持续激励玩家完成更多邀请里程碑。
旅程组件配置
  • 开始组件：属性: 已完成身份认证，邀请人数 ≥ 1 触发进入
  • 监听事件（多分支）：事件"好友注册"，多路分流
  • 分支 1（达标通知）：邀请人数 = 5/20/100/200/500/1000 → 即时发送"领取奖励"通知
  • 分支 2（即将达标预警）：邀请人数 = 3/18/98/198/498/998 → 发送"还差 2 人即可领取下一奖励"激励消息
  • 转化统计：事件"好友注册"，监听 7 天
  • 再次进入：2 分钟后可再次进入，不限制次数
可参考指标
进度提醒驱动邀请完成率提升，配合奖励裂变模型，新增注册预期提升 15–25%。

六、合规与数据安全

数据安全
  • 全链路 TLS 加密，传输层与存储层数据分离
  • VPC 网络隔离，支持 IP 白名单访问控制
  • PII 数据脱敏处理，用户标识与业务数据分层存储
  • RBAC 角色权限控制，操作日志完整留存
  • 数据中心可按客户安全需求灵活扩展
认证与合规
  • 已通过 GDPR 与 SOC 2 认证
  • 可协助客户满足各目标市场本地数据保护法规要求
  • 消息发送记录保留 60 天
  • 发送前支持内容样式预览
运营护栏
  • 频控策略：旅程级别限制每日/每周消息数量，避免过度打扰
  • 勿扰时段：可发送时间组件 + 终端时区匹配，确保在合适时间触达
  • 提前退出：用户完成目标行为后自动退出旅程，不再发送后续消息
  • 全渠道退订管理，符合各平台推送规范
渠道合规
  • WhatsApp：基于 Meta 官方审批模板发送，合规触达
  • App Push：遵循 Apple/Google 推送规范，支持权限引导
  • SMS：对接本地合规运营商，支持 Sender ID 注册
  • Email：SPF/DKIM/DMARC 全配置，提升送达率与安全性

七、4 周测试计划

1第 1 周 · 对齐与埋点
  • 明确数据打通优先级与关键事件清单
  • SDK 集成：注册、身份认证、充值、登录等核心事件
  • 用户属性同步：VIP 等级、累计充值、最后登录时间等
  • 确认推送渠道：App Push 证书 + WhatsApp 模板审批
2第 2 周 · 旅程构建
  • 核心旅程全量上线：新用户激活 + 充值挽回 + 沉睡唤醒
  • 旅程配置审核：进入条件、分支逻辑、频控规则
  • A/B 文案测试：首充引导 vs 活动激励文案对比
  • 发送时间验证：终端时区设置与本地时段覆盖
3第 3 周 · 数据验证
  • 查看各旅程转化漏斗与消息点击率
  • 对比 MA 触达 vs 自然转化的效果差异
  • 验收指标：身份认证率、首充转化率、7 日留存率
  • 发现并修正配置问题（进入条件、等待时长等）
4第 4 周 · 复盘扩面
  • 复盘 3 周数据，提炼高效场景，优化低效旅程
  • 扩展场景：VIP 升级通知 + 每周返利提醒 + 邀请好友
  • WhatsApp / SMS 渠道补量，覆盖 Push 未触达用户
  • 制定持续运营路线图，规划后续季度旅程扩展方向
