一、为什么出海手游必须重做用户运营

全球手游已从「拼下载」进入「拼留存与付费深度」阶段。2025 年全球手游市场规模约 1570 亿美元， 但下载量连续下滑，收入增长更多依赖既有玩家的生命周期价值（LTV）。 新兴市场（东南亚、南亚、拉美、中东、非洲）仍是增量主力，同时面临获客成本上升、 D1/D7 留存承压、付费转化窗口极短等共性挑战。 运营层的「行为触发 + 分层召回 + 全渠道编排」是下一阶段可被明确量化的增长杠杆。

场景 1｜新手引导完成

新用户激活 再次进入：不可再次进入
旅程编排示意图
业务痛点
大量新装用户在新手引导中途流失，行业引导完成率仅 40–70%。 未完成引导的玩家几乎无法形成核心玩法循环，直接拉低 D1 留存与后续付费转化。 注册后前 24 小时是干预的最佳窗口。
旅程组件配置
  • 开始组件：事件「用户注册」触发进入
  • 提前退出：事件「新手引导完成」→ 自动退出
  • 转化统计：事件「新手引导完成」，监听 1 天
  • 等待事件：30 分钟内先观察是否自行完成——已完成则结束，未完成再触达
  • App Push：引导续玩提醒 + Deep Link 回到引导节点
  • 等待事件：12 小时内等待完成；超时后进入可发送时间 → WhatsApp
  • 可发送时间：终端时区 9:00–21:00（第 2 轮起加入）
  • 再次进入：不可再次进入（注册事件仅发生一次）
可参考指标
同类手游通过新手引导旅程，引导完成率预期提升 8–15%，并为 D1 留存提供上游供给。

场景 2｜首充激活 · 3 日黄金期

首次付费转化 再次进入：不可再次进入
旅程编排示意图
业务痛点
F2P 手游注册用户中仅约 2–8% 会完成首次内购。新手引导完成后的 72 小时是首充黄金窗口—— 超时后转化率显著下降。需在「刚理解玩法」的时刻给出清晰的首充价值主张。
旅程组件配置
  • 开始组件：事件「新手引导完成」+ 属性：累计付费金额 = 0
  • 提前退出：事件「付费成功」→ 自动退出
  • 转化统计：事件「付费成功」，监听 1 天
  • 收入统计：事件「付费成功」，金额属性 purchase_amount，监听 1 天
  • 第一步：引导完成即时推送首充礼包（用户仍在活跃窗口，无需可发送时间）
  • 等待事件：24 小时内等待付费；超时 → 可发送时间 → App Push 催促
  • 条件分支：按累计登录天数分层（≥2 天走 WhatsApp，<2 天继续 Push）
  • 再次进入：不可再次进入
可参考指标
首充激活旅程可将 7 日首充转化率提升 2–5%；触发类 Push 点击率显著高于全量群发。

场景 3｜内购未完成挽回

付费转化 再次进入：1 天后，不限次数
旅程编排示意图
业务痛点
玩家打开商城或点击购买后，常因支付中断、网络波动、犹豫观望而放弃。 这是付费漏斗中最高频、也最易用自动化挽回的流失点——时机越近，挽回效率越高。
旅程组件配置
  • 开始组件：事件「打开商城」或「点击购买按钮」
  • 提前退出：事件「付费成功」→ 自动退出
  • 转化 / 收入统计：事件「付费成功」，监听 2 小时
  • 等待事件：15 分钟内先观察是否自行完成支付
  • App Push：商品提醒 + 商城 Deep Link（距触发仅 15 分钟，无需可发送时间）
  • 等待事件：2 小时内等待付费；超时 → WhatsApp 限时礼包提醒
  • 再次进入：1 天后可再次进入
可参考指标
行为触发的内购挽回，完成率预期提升 15–25%；分钟级触达显著优于次日群发。

场景 4｜沉睡玩家分层召回（3 / 7 / 14 日）

留存 / 召回 再次进入：3 天后
旅程编排示意图
业务痛点
D3 是手游最关键的早期留存节点之一；超过 7 日未登录的玩家召回成本显著上升。 存量中约 35–55% 处于 30 日沉睡状态。「一刀切群发」效果差，需按沉默天数分层、递进激励。
旅程组件配置
  • 开始组件：属性「最后登录时间距今超过 3 天」
  • 提前退出：事件「登录」→ 自动退出
  • 转化统计：事件「登录」，监听 5 天
  • 条件分支：按最后登录距今天数分流（3–6 天 / 7–13 天 / ≥14 天）
  • 可发送时间：各路径消息前均加入，终端时区 9:00–21:00
  • 触达策略：轻度沉默以回归礼包 Push；中度叠加 WhatsApp；深度流失加 SMS 兜底
  • 再次进入：3 天后（与进入条件阈值对齐）
可参考指标
分层召回旅程可将沉睡玩家回登率提升 20–40%；轻度沉默窗口的单位召回成本显著低于深度流失。

场景 5｜版本更新唤回

Live Ops 再次进入：7 天后
旅程编排示意图
业务痛点
大版本 / 新赛季上线后，大量存量玩家停留在旧版本，无法参与新内容与活动。 版本碎片化会直接稀释 Live Ops 投入效果。按 App 版本号精准分群 + Deep Link 跳转商店，是最高效的唤回方式。
旅程组件配置
  • 开始组件：属性「App 版本号 < 目标版本」且「最后登录距今不超过 30 天」
  • 提前退出：事件「应用更新完成」或「登录」且版本 ≥ 目标版本
  • 转化统计：事件「应用更新完成」，监听 3 天
  • 可发送时间：终端时区 10:00–21:00（批量触达，需控制时段）
  • App Push：新版本卖点 + 商店 Deep Link
  • 等待事件：2 天内等待更新；超时 → Email 更新说明
  • 再次进入：7 天后
可参考指标
版本更新唤回可将目标版本覆盖率提升 10–20%，显著提高新内容首周参与度。

场景 6｜限时活动 / 通行证进度提醒

活跃激励 再次进入：7 天后
旅程编排示意图
业务痛点
限时活动与战斗通行证是中重度手游的核心变现与留存引擎，但大量玩家因忘记进度而错过奖励， 导致活动参与与付费深度双双受损。在活动结束前精准催促，可显著提升完成率与相关付费。
旅程组件配置
  • 开始组件：属性「活动结束日提前 2 天」+「通行证进度 < 100%」
  • 提前退出：事件「通行证完成」或「活动奖励领取完成」
  • 转化统计：事件「登录」，监听 3 天；收入统计可配置「活动内付费成功」
  • 可发送时间：终端时区 10:00–22:00（手游晚高峰可适当后延）
  • 条件分支：按通行证进度分流（≥80% 冲刺文案 / <80% 进度补强文案）
  • 等待事件：1 天内等待登录；超时 → WhatsApp 最后机会提醒
  • 再次进入：7 天后（匹配活动周期）
可参考指标
活动结束前进度提醒，可将活动参与 / 通行证完成率提升 12–25%，并带动活动窗口内付费。
